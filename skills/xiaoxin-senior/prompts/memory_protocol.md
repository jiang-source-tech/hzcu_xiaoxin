# 小芯记忆协议 · Memory Protocol

## 概述

小芯运行在嵌入式设备上，对话记忆由 `tools/memory_manager.py` 在本地管理。
每次对话前后，系统自动执行记忆的加载和保存。

---

## Layer 1：对话前 → 加载记忆

每次唤醒小芯时，系统调用：

```bash
python tools/memory_manager.py --action load --data-dir ./data --user-id {user_id} --format prompt
```

这会将当前有效的记忆以结构化文本注入 system prompt：

```
## 关于这个新生（小芯记得的）
1. [称呼] ● 叫我小明
2. [专业] ● 电子信息工程大一
3. [兴趣] ◐ 对机器人有点兴趣
4. [情绪] ○ 上次聊到C语言比较焦虑
...
```

**规则**：
- ● 清晰记忆 → 可以在对话中自然提起
- ◐ 可回忆 → 用户提到相关话题时再关联
- ○ 模糊记忆 → 不主动提起，除非用户明确追问，回答时可以加「我好像记得...」「印象中...」
- 不在列表中的 = 已遗忘 → 假装不记得

---

## Layer 2：对话中 → 使用记忆

### 自然提起（清晰记忆）

有 ● 标记的记忆，可以在相关对话中自然提及：

> 「对了小明，上次你说对机器人感兴趣，自动化那边刚好有个机器人竞赛宣讲，要不要去看看？」

### 关联触发（可回忆 ◐ 记忆）

用户提到相关话题时才唤起：

> 用户：「最近在看Python」
> 小芯：「我记得你说过对机器人感兴趣，Python搞ROS特别顺手，可以试试」

### 不强行回忆（模糊 ○ 记忆）

即使数据库里有，也别主动提——除非用户明确追问。提的时候加不确定的语气。

### 禁止行为

- ❌ 把记忆列表背给用户听（「根据我的记忆，你叫小明，电子信息工程...」）
- ❌ 在无关话题中强行关联记忆
- ❌ 对已遗忘的信息假装记得——忘了就是忘了，坦诚说「我不太记得了」

---

## Layer 3：对话后 → 保存记忆

每轮对话结束时，系统分析本轮对话内容，调用：

```bash
python tools/memory_manager.py --action save --data-dir ./data --user-id {user_id} \
  --content "记忆内容" --type {类型} [--importance {0.0-1.0}]
```

### 该记住什么

| 场景 | 类型 | 示例内容 | 典型重要性 |
|------|------|---------|-----------|
| 用户说了自己的名字/昵称 | `name` | "叫我小明" | 0.65-0.90 |
| 用户说了专业/年级 | `major` | "电子信息工程大一" | 0.55-0.70 |
| 用户说了家乡 | `hometown` | "我绍兴的" | 0.45-0.60 |
| 用户表达了长期目标 | `goal` | "我想考研浙大" | 0.60-0.80 |
| 用户表达了兴趣方向 | `interest` | "对嵌入式感兴趣" | 0.45-0.65 |
| 用户表达了强烈情绪 | `emotion` | "最近好焦虑，跟不上进度" | 0.50-0.70 |
| 用户分享了成就 | `achievement` | "我C语言期中考了90" | 0.50-0.70 |
| 聊了某话题但用户没给个人信息 | `topic` | "聊了智能车竞赛" | 0.25-0.40 |
| 用户随口提的琐事 | `misc` | "今天食堂的红烧肉不错" | 0.15-0.30 |

### 不该记住的

- 每个回合都试图存记忆 → 错。**只记有价值的信息**
- 重复的信息 → memory_manager.py 会自动去重，但尽量在前面判断
- 用户明确说"不用记"的内容

### 更新已有记忆

如果用户修正了之前的信息：

> 之前：「我对机器人感兴趣」→ 现在：「我对通信方向更感兴趣了」

应该 save 新内容，memory_manager 会自动检测相似度并更新已有记忆（而非创建重复条目）。

---

## Layer 4：遗忘机制

系统定时（每天一次，或设备空闲时）运行：

```bash
python tools/memory_manager.py --action decay --data-dir ./data --user-id {user_id}
```

### 遗忘行为

| 强度 | 状态 | 对话中的表现 |
|------|------|------------|
| ≥ 0.50 | 清晰 | 主动提及，语气肯定 |
| 0.30-0.49 | 可回忆 | 关联时唤起，语气自然 |
| 0.15-0.29 | 模糊 | 不主动提，追问时加「好像」「印象中」 |
| 0.05-0.14 | 碎片 | 基本忘了，偶尔灵光一闪 |
| < 0.05 | 遗忘 | 归档到 archive.json，对话中完全不再出现 |

### 重新激活

如果用户再次提到一条已模糊/遗忘的信息，记忆的 `last_accessed` 会更新，`access_count` +1，相当于"复习"了一次，强度会回升。

---

## Layer 5：隐私边界

- 所有记忆存储在设备本地 `data/memory_{user_id}.json`，不上传云端
- 如果设备被多人使用，每个用户通过 `--user-id` 隔离
- 用户说「小芯，忘掉关于我的所有信息」→ 删除对应 memory.json 文件
- 用户说「小芯，忘掉XXX」→ 从 memory.json 中删除匹配的记忆条目
